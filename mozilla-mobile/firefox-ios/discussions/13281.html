    I quickly hacked something into the code and it works correctly. The important point is: "A WKWebView is included in the array of activityItems".

    So Just as a POC:

    in ShareExtensionHelper: modify createActivityViewController to accept a webView as a param and to add it to the activity items:

    func createActivityViewController(webView: WKWebView? = nil, _ completionHandler: @escaping (_ completed: Bool, _ activityType: UIActivity.ActivityType?) -> Void) -> UIActivityViewController {
    var activityItems = getActivityItems(url: url)
    if webView != nil {activityItems.append(webView!)}

    in BrowserViewController presentShareSheet: call createActivityViewController to pass in the tab's webview:

        let controller = helper.createActivityViewController(webView: tab?.webView, { [unowned self] completed, activityType in

    That's all that's needed When you run it on ios16.4 the Add to Home Screen option is available and works.

    Notes:

    1. Running this in Fennec won't work due to the lack of com.apple.developer.web-browser entitlement, but in Firefox it does.
    2. In the simulator, the option to add to the home screen shows up and 'works' except nothing actually gets added to the Home Screen. I suspect this is an imitation of the simulator as the same happens in Safari. All is goos on a real device.
    3. There is no great need to wait for iOS 16.4 to be released before implementing this as pre-16.4 the Webview added to the activityItems sits dormant doing nothing. You could wrap the adding to the list with a #available(iOS 16.4), but its not needed.
    4. This doesn't;t need the iOS 16.4 SDK nor Xcode 14.3 beta to do this.
    5. I didn't attempt to add a webview to any other calls to createActivityViewController. I just wanted to know if it would work and wasn't sure when all the other calls were made.

    It's not automatically added to the Share menu. Browsers on iOS will need to add support.

    Applications on iOS and iPadOS present the Share menu by creating a UIActivityViewController with an array of activityItems. For “Add to Home Screen” to be included in the Share menu the following must be true:

    • The application has the com.apple.developer.web-browser managed entitlement
    • A WKWebView is included in the array of activityItems
    • The WKWebView is displaying a document with an HTTP or HTTPS URL
    • If the device is an iPad, it must not be configured as a Shared iPad

    We are happy to help if there are any questions.
